Parameters injection

For benchmarking setting, users can change the LoRa radio parameter value without modifying the code. This is achieved by the method similar used in D-Cube.
Example project is detailed here.
Project configuration:
1. As procedures in firmware preparation, open Project->Properties, go to Paths and Symbols in C/C++ General, add ChirpBox/Miscellaneous/Parameters-patch/Inc/ to the include directories.
2. Add following codes in your main.c file.

3. Add following GCC linker flags to locate the parameter section address.
-Wl,--section-start -Wl,.FUTSettingSection=0x08000610
-Wl,--section-start -Wl,.text=0x08000620

4. Compile project and generate the .bin file.

Firmware patch with parameters:
5. Config parameters in ChirpBox/chirpbox manager/Tools/param_patch/fut-param-settings.json. For example, we set the LoRa working on 470 MHz with spreading factor 7, TX power as 14 dBm and 4:5 (4-bit data with 1-bit redundancy) coding rate:


6. Run fut_generate tool under ChirpBox/chirpbox manager/ to update the specified FUT firmware.
